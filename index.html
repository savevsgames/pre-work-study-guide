HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML stands for Hypertext Markup Language and is the standard markup language for creating web pages.
  • The !DOCTYPE html declaration defines that this document is an HTML5 document.
  • The html element is the root element of an HTML page.
  • The head element contains meta information about the HTML page.
  • The title element specifies a title for the HTML page, which is shown in the browser's title bar.
  • The body element defines the document's body, which contains all the visible contents, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.
  • HTML headings are defined with the h1 to h6 tags, where h1 is the most important and h6 is the least important.
  • HTML links are defined with the a tag and the href attribute specifies the URL of the page the link goes to.
  • HTML images are defined with the img tag, and the src attribute specifies the path to the image.
  • HTML lists can be unordered ul or ordered ol, and list items are defined with the li tag.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS stands for Cascading Style Sheets and is used to style and layout web pages.
  • The box model in CSS consists of content, padding, border, and margin.
  • Flexbox is a one-dimensional layout method for arranging items in rows or columns.
  • CSS Grid is a two-dimensional layout system that allows for complex layouts with rows and columns.
  • The 'z-index' property controls the stacking order of elements on a web page.
  • Pseudo-classes in CSS are used to define a special state of an element, like :hover or :focus.
  • Media queries allow you to apply CSS styles based on device characteristics, like screen size.
  • CSS variables, also known as custom properties, let you store specific values to reuse throughout a document.
  • The 'transition' property allows you to change property values smoothly over a specified duration.
  • CSS animations allow you to create keyframes to animate elements without using JavaScript.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Initialize a new repository: git init
  • Clone an existing repository: git clone https://github.com/username/repository-name.git
  • Check the status of your repository: git status
  • Add files to the staging area: git add filename.txt (for a specific file) or git add . (for all files)
  • Commit changes: git commit -m "Your commit message here"
  • Push changes to GitHub: git push origin main
  • Pull changes from GitHub: git pull origin main
  • Create a new branch: git branch branch-name
  • Switch to a different branch: git checkout branch-name
  • Merge branches: git merge branch-name
  • Create a new repository on GitHub: Click the "+" icon in the top right corner and select "New repository"
  • Fork a repository: Navigate to the repository you want to fork and click the "Fork" button in the top right corner
  • Create a pull request: After making changes in a forked repository, click on "Pull requests" and then "New pull request"
  • Use GitHub Issues: Navigate to the "Issues" tab in a repository to create, assign, or comment on issues
  • Edit files directly on GitHub: Click on a file in your repository, then click the pencil icon to edit it directly in the browser
  • Use GitHub Pages: Enable GitHub Pages in your repository settings to host a website directly from your GitHub repository
  • Explore GitHub's social features: Follow other users, star repositories you find interesting, and contribute to open-source projects
  • Use GitHub's search functionality: Utilize the search bar at the top of GitHub to find repositories, users, or specific code snippets
  • Review your repository's insights: Click on the "Insights" tab in your repository to view statistics and analytics about your project
  • Utilize GitHub's project management tools: Use the "Projects" tab to create kanban boards and manage your project's workflow

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript was created in 1995 by Brendan Eich while he was an engineer at Netscape.
  • JavaScript is a high-level, interpreted programming language that is primarily used for web development, creating dynamic websites, and adding interactivity to web pages.
  • JavaScript is a client-side scripting language, which means it is executed on the user's web browser, rather than on the server.
  • JavaScript is an object-oriented language, with objects, properties, methods, and events.
  • JavaScript can be used to validate form data, create cookies, and detect the user's browser.
  • JavaScript can be used to create interactive effects, such as drop-down menus, image sliders, form validations, and pop-ups.
  • JavaScript is an interpreted language, which means it is executed line by line, without the need for compilation.
  • JavaScript is a case-sensitive language, which means that "myFunction()" and "myfunction()" are two different functions.
  • JavaScript can be embedded directly into HTML pages or stored in external .js files and linked to from the HTML page.
  • JavaScript is supported by all modern web browsers, including Google Chrome, Mozilla Firefox, Safari, Opera, and Microsoft Edge.
