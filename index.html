HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The <!DOCTYPE html> declaration defines the document type and version of HTML.
  • The html element is the root element of an HTML page.
  • The title element specifies the title of the webpage shown in the browser's title bar or tab.
  • The meta element provides metadata about the HTML document, such as character set and viewport settings.
  • The h1 to h6 elements represent different levels of headings, with h1 being the highest level.
  • The p element represents a paragraph of text.
  • The a element defines a hyperlink to link one page to another.
  • The img element is used to embed images in a webpage.
  • The ul and ol elements are used to create unordered and ordered lists, respectively.
  • The div element is a container used to group and style sections of a webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A selector is a pattern used to select the element(s) you want to style.
  • A property is a style attribute you want to change, such as color, font-size, or margin.
  • The box model consists of margins, borders, padding, and the actual content of an element.
  • The display property specifies if/how an element is displayed and includes values like block, inline, and none.
  • Flexbox is a layout model that allows items within a container to be aligned and distributed.
  • A CSS class is a reusable style definition that can be applied to multiple elements.
  • IDs are unique style identifiers that can be applied to a single element per page.
  • Media queries allow for responsive design by applying different styles for different device sizes.
  • The position property specifies how an element is positioned in the document, with values like static, relative, absolute, and fixed.
  • CSS animations allow elements to transition between styles over a specified duration.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Initialize a new repository: git init
  • Clone an existing repository: git clone https://github.com/username/repository-name.git
  • Check the status of your repository: git status
  • Add files to the staging area: git add filename.txt (for a specific file) or git add . (for all files)
  • Commit changes: git commit -m "Your commit message here"
  • Push changes to GitHub: git push origin main
  • Pull changes from GitHub: git pull origin main
  • Create a new branch: git branch branch-name
  • Switch to a different branch: git checkout branch-name
  • Merge branches: git merge branch-name
  • Create a new repository on GitHub: Click the "+" icon in the top right corner and select "New repository"
  • Fork a repository: Navigate to the repository you want to fork and click the "Fork" button in the top right corner
  • Create a pull request: After making changes in a forked repository, click on "Pull requests" and then "New pull request"
  • Use GitHub Issues: Navigate to the "Issues" tab in a repository to create, assign, or comment on issues
  • Edit files directly on GitHub: Click on a file in your repository, then click the pencil icon to edit it directly in the browser
  • Use GitHub Pages: Enable GitHub Pages in your repository settings to host a website directly from your GitHub repository
  • Explore GitHub's social features: Follow other users, star repositories you find interesting, and contribute to open-source projects
  • Use GitHub's search functionality: Utilize the search bar at the top of GitHub to find repositories, users, or specific code snippets
  • Review your repository's insights: Click on the "Insights" tab in your repository to view statistics and analytics about your project
  • Utilize GitHub's project management tools: Use the "Projects" tab to create kanban boards and manage your project's workflow

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is a high-level, interpreted programming language commonly used in web development.
  • Variables can be declared using the keywords var, let, or const.
  • Functions are reusable blocks of code that perform a specific task.
  • Objects are collections of key-value pairs that represent real-world entities.
  • Arrays are ordered lists of values that can be accessed by their index.
  • Loops allow us to execute a block of code multiple times.
  • Conditionals allow us to execute different blocks of code based on certain conditions.
  • Scope determines the accessibility of variables in different parts of the code.
  • Events are actions or occurrences that happen in the browser, which can be detected and handled using event listeners.
  • Closures are functions that have access to variables from another function's scope.
